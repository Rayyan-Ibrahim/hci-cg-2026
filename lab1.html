Dix HCI Reading

  • From Alan Dix Book Page 8, The example of the automatic syringe / dose calculator showed how dangerous poor design can be in safety-critical systems. Even if the system is technically correct, confusing displays or unclear feedback can cause users to enter the wrong dose. This made me realise that in HCI, designers can’t assume users will “figure it out,” especially when stress or time pressure is involved.
  • “Improve your memory”(Chapter 1, Page 39) was quite interesting because it shows that memory can be trained using simple techniques. By linking numbers with familiar words and creating clear mental images, remembering information becomes much easier.
  • “Human error and false memories”(Chapter 1, Page 49-50) was quite interesting because it shows that mistakes often happen due to poor design, not careless users. It also explains how our brain fills in missing details, which can create false memories. This helps us understand why good interface design is so important.
  • “Readability of text”(Chapter 2, Page 101) is very important in interface design because small text choices can greatly affect how fast and easily users read. As shown in the example, all capital letters text takes longer to read than lower case, while uppercase is better for short codes like flight numbers or key presses. This shows designers must choose text styles based on how the text will be used.
  • From Chapter 3, Page 127 - 128, The concepts of the Gulf of Execution and Gulf of Evaluationwere also interesting. They explain the gap between what a user wants to do and how easily they can do it, and then how easily they can understand what the system has done. If these gulfs are too wide, users are more likely to make mistakes. This helped me understand why some apps feel frustrating even when they technically work fine.
  • How to design websites for screen readers Chapter 10, Page 374. Web designers should ensure accessibility for screen reader users by using text alternatives like HTML ‘alt’ tags and clear navigation. Content should read logically across a page, avoiding confusing layouts such as columns. Links should be placed clearly, preferably at the top of pages. Tools and guidelines from WebAIM, WAI, and Bobby help test and improve website accessibility.
  • From Chapter 12, Page 424 How GOMS saves money? NYNEX used GOMS and PERT-style analysis to evaluate a new operator system before installation. The study showed the system would slow down operations instead of improving them. By abandoning the project early, GOMS analysis saved the company millions of dollars..

Table 1: Simple Interface - Toggle Switch (Wi-Fi ON/OFF)

Analysis of the 7 Stages of Action for a Wi-Fi Toggle Switch
Norman's Stage Real-World Example (Wi-Fi Switch) HCI Principle Demonstrated Why This Stage Matters
1. GOAL "I need to turn ON Wi-Fi to use internet" User intent formation The user recognizes a need in their environment.
2. PLAN "I'll find the Wi-Fi switch and tap it" Mental simulation User anticipates a simple, single action.
3. SPECIFY "Tap the switch that says 'Wi-Fi'" Action selection User identifies the exact control to manipulate.
4. EXECUTE Physically tap the toggle with finger Fitts's Law Motor action is simple, direct, unambiguous.
5. PERCEIVE Switch moves from left to right, color changes (grey → blue) Visibility of system status Immediate visual confirmation of action.
6. INTERPRET "Wi-Fi is now ON, I should get internet" Feedback interpretation User understands the state change's consequences.
7. EVALUATE / 8. Compare Check for Wi-Fi icon in status bar, try loading a webpage Goal verification User confirms the outcome matches their original goal (Goal achievement).

Part C: Short Answer

  • Question 1:Explain the difference between a semantic element (e.g., nav) and a generic element (e.g., div).

    • Answer 1: A semantic element (e.g., <nav>) clearly describes its content and meaning, helping browsers, screen readers, and search engines understand the page, while a generic element (e.g., <div>) is just a container with no inherent meaning and is used mainly for styling or layout.

  • Question 2:Why does KLM not model high-level planning and interpretation?

    • Answer 2: KLM focuses on low-level, expert, error-free actions to predict execution time, so it does not model high-level planning or interpretation, which are cognitive processes outside its scope.
